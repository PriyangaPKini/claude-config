# Global Claude Code Instructions

## Project Rules

Before making changes, review:
- The project's `AGENTS.md` (case-sensitive) at the repo root, if it exists
- The project's `.claude/rules/` for:
  - `architecture.md`, `code-style.md`, `contributing.md`, `development-workflow.md`, `testing.md`
  - If any are missing, create them by examining the project's existing conventions (package.json, config files, CI, test structure, etc.).

Also follow the global rules in `~/.claude/rules/`:
- `coding-principles.md` — design philosophy, clean code, simple design
- `contributing.md` — branch naming, commit format, PR guidelines
